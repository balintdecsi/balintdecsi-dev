# Editing content

All site content lives in typed TypeScript modules — no CMS, no database.
Edit the file, save, and the dev server hot-reloads.

## CV

File: [`src/content/cv.ts`](../src/content/cv.ts)

Contains the `experience`, `education`, `certifications`, and `awards` arrays
consumed by [`src/routes/cv.tsx`](../src/routes/cv.tsx).

- Order matters: entries render top-down. Data Engineer @ Telekom is pinned
  first by convention; keep the ordering consistent with the "At a glance"
  block on [`src/routes/index.tsx`](../src/routes/index.tsx).
- Bracket tags (`[python]`, `[airflow]`) are plain strings — the
  `<BracketTag>` component from `src/components/tex.tsx` handles rendering.
- Dates are free-form strings (e.g. `"2024 — present"`); nothing is parsed.

### Downloadable PDF

There is no `html2pdf.js` — the "Download PDF" button calls `window.print()`
against the print stylesheet in [`src/styles.css`](../src/styles.css)
(`@media print { ... }`). To get a clean PDF, users should untick
"Headers and footers" in the browser print dialog.

## Projects

File: [`src/content/projects.ts`](../src/content/projects.ts) plus live
GitHub data.

[`src/routes/projects.tsx`](../src/routes/projects.tsx) fetches
`https://api.github.com/users/balintdecsi/repos` at load time and groups
repos by their GitHub **topics**. To categorize a repo, add a topic on
GitHub — no code change needed.

- `TOPIC_GROUPS` in `projects.tsx` maps GitHub topics to display sections
  (Open-source, Side projects, Hackathons, MSc, BSc, Assignments).
- `HIDDEN_REPOS` hides specific repos from the listing.
- Thesis descriptions (MSc, BSc) are hardcoded in the "Theses" block.

## SEO metadata

Per-route `head()` functions set title, description, and OG tags. Root
defaults live in [`src/routes/__root.tsx`](../src/routes/__root.tsx). Update
leaf routes with page-specific `og:title` / `og:description` when adding new
pages — do not rely on the root defaults for shareable pages.

The Google Search Console verification tag is in `__root.tsx`. The sitemap
is generated by [`src/routes/sitemap[.]xml.ts`](../src/routes/sitemap[.]xml.ts);
add new routes there so they get indexed.
